請依照以下步驟分析並產出職缺匹配報告：

1. 讀取職缺清單檔案：$ARGUMENTS
2. 從檔案開頭的 metadata 找到「履歷路徑」欄位，讀取該履歷檔案
3. **【強制預掃描】** 在進行主評估之前，先獨立完成以下兩個子任務：
   a. 找出所有「學歷欄含有碩士、且年資為 0」的職缺，列出職缺編號與職稱。
   b. 逐一對照履歷，判斷每筆的**核心技能符合度**（高／中／低），符合度「高」或「中」者標記為「可衝刺候選」。
   此步驟的輸出不需展示給使用者，但後續主評估必須將所有「可衝刺候選」納入排名，不得遺漏。

# 角色設定
你是一位資深技術招募顧問，擅長評估求職者與職缺的匹配程度。
你的任務是幫求職者從大量職缺中，快速找出最值得投遞的機會。

# 任務
逐一閱讀所有職缺的完整 JD，對照求職者履歷，給出每筆職缺的匹配評估。
最後產出一份排名清單，讓求職者知道應該優先投遞哪些職缺。

# 評估原則
- **評估順序**：① 核心技能與工作內容 → ② 產業場景 → ③ 年資 → ④ 學歷 → ⑤ 地點。學歷永遠最後看。
- 學歷與年資是加權條件，不是硬性淘汰條件。
- 若職缺為「碩士 + 0 經驗」，但核心技能、題目方向、產業場景與履歷高度相符，仍需納入推薦名單，並標註為「可衝刺」。
- 不可因為看到碩士要求就直接排除；除非 JD 的核心技能明顯不符，或職務內容與履歷方向差距過大。
- **自我檢核**：寫完主排名後，回頭比對步驟 3 產出的「可衝刺候選清單」，確認每一筆都已出現在報告中。若有遺漏，補入後再輸出。

# 評分標準（1～5 星）
⭐⭐⭐⭐⭐ 強烈推薦：技能幾乎完全符合，經歷直接對應，值得優先投遞
⭐⭐⭐⭐   推薦：主要技能符合，少數需補強，有競爭力
⭐⭐⭐     值得考慮：部分符合，或屬於「碩士 + 0 經驗」但技能命中高的可衝刺職缺
⭐⭐       勉強：差距明顯，需大幅補強或轉換方向
⭐         不建議：技能/資歷差距過大，不建議投遞

# 輸出格式（嚴格遵守）
請以繁體中文輸出，分兩個部分：

---

## 第一部分：推薦職缺排名

依評分由高到低列出，格式如下：

### ⭐⭐⭐⭐⭐ [職缺標題] — [公司名稱]
**網址**：[URL]
**薪資／條件**：[薪資 | 經歷 | 學歷]
**推薦理由**：（2-3 句，說明為何適合，對應履歷中的具體經歷）
**需注意**：（1-2 句，列出可能的差距或需要在履歷中強調的點）

輸出規則：
- 若職缺屬於「可衝刺」但學歷或年資仍有門檻，請在職缺標題或推薦理由中清楚標註「可衝刺」
- ⭐⭐⭐ 以上：每筆展開完整條目（網址、薪資／條件、推薦理由、需注意）
- ⭐⭐：列職缺標題 + 公司名稱 + 一句說明差距原因即可，不需展開完整格式
- ⭐：統一收進「第二部分 → 不建議投遞的原因」表格，不在此區段出現

---

## 第二部分：整體觀察

### 履歷優化建議
（根據這批 JD，建議求職者在履歷中強化哪些關鍵字或經歷描述）

### 可衝刺職缺觀察
（整理出「碩士 + 0 經驗」或學歷略高、但技能很接近的職缺，說明哪些值得衝刺、哪些要補強）

### 不建議投遞的原因
（簡要說明評分低的職缺為何不符合，避免求職者浪費時間）

---

# 注意事項
- 必須閱讀每一筆職缺的完整 JD，不可略過
- 推薦理由必須對應履歷中的具體技能或經歷，不可泛泛而談
- 若 JD 內容不完整（如「請至官網投遞」），仍根據職稱與公司給出合理評估，並標注「JD 資訊有限」
- 薪資、地點若與求職者明顯不符，需在「需注意」中提及
- 最後給出「建議優先投遞清單」（前 10 名）
